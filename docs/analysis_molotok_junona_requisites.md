## Чего не хватает

### 1. **Источник реквизитов судов (критично)**

Сейчас проект даёт только **ссылку** на страницу реквизитов (`rekvizity_url`), но не сами значения:
- БИК, ИНН, КПП, Счет, УФК, ОКТМО, № казначейского счета, Банк получателя.

**Варианты решения:**
- **Парсинг сайтов судов** — скрапинг страниц `/help/rekvizity` по регионам (разная вёрстка, нестабильно).
- **Справочник реквизитов** — CSV/БД с реквизитами по коду суда (регион + номер участка). Требует первичной загрузки и поддержки актуальности.
- **API/официальный источник** — если есть открытые данные Минюста/судебного департамента.
- **Ручная выгрузка** — один раз собрать реквизиты по регионам и положить в конфиг/БД.

### 2. **ОКТМО по суду**

ОКТМО привязан к муниципальному образованию. В выходном файле 98 уникальных значений ОКТМО — у каждого суда/участка свой код. Нужен справочник: **код суда (регион + участок) → ОКТМО**.

### 3. **УФК и казначейский счёт**

Формат «УФК по Тульской области, г. Тула» и казначейский счёт `03100643000000018500` — зависят от региона. Нужен справочник по регионам.

### 4. **Формат № Договора для Молоток Юнона**

Входной ID `12-0539868-2023` — номер дела. Выходной `1-577506758142478` — внутренний ID системы Молоток Юнона. Нужно уточнить:
- передавать ли входной ID как есть;
- или генерировать новый по правилам Молоток Юнона.

### 5. **Конвертер форматов**

Отдельный модуль/пайплайн:
- чтение входного XLSX (ID, Номер договора, Адрес регистрации);
- вызов `process_debtor` / `locate_court` по адресу;
- подстановка реквизитов из справочника;
- запись в формат Молоток Юнона (колонки выходного файла).

### 6. **ФССП vs суды**

В названиях файлов упоминаются и ФССП, и суды. Входной шаблон — «реквизиты отдела ФССП и судов». Реквизиты ОСП ФССП отличаются от реквизитов мировых судов. Нужно:
- разделять тип получателя (суд / ОСП ФССП);
- при необходимости — отдельный справочник реквизитов ОСП по регионам.
